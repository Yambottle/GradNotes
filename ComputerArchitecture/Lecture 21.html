1. larger memory is slower:
    1. longer wires
    2. bigger address decoding
  2. upper level caches are included in the lower level or separated, book assumes include
    1. share same memory location and state 
  3. Locality:
    1. Spatial:
      1. close enough to be used frequently
    2. Temporal:
      1. use it now, probably reuse it again
  4. Direct mapped: fast, but collision or cache miss
    1. One 8bytes word one line of cache
      1. For 8 bits address
      2. Tag is first three bits
      3. After three bits show which row in cache line
      4. Last two bits show which byte in the word
    2. two 4bytes words shared one line of cache
      1. For 8 bits address
      2. Tag is first three bits
      3. After two bits show which row in cache line
      4. The following one shows which word in block(divide previous into two words, so need one bit to show which block is)
      5. Last two bits show which byte in the word
    3. compulsory miss: first access to that block
    4. Number of rows in cache = # of words that can be held by cache/ # of words per block
    5. For 32 bits address
      1. 31-14: tag
      2. 13-8:  row in cache
      3. 7-2: word in block
      4. 1-0: byte in word
  5. Fully associative: cache can be anywhere; reduce miss rate, but increase access time
    1. tag gets larger because there are fewer bits that say where to look, don’t need show which row
    2. For 8bits address
      1. last two which byte in word
      2. last third which word in block
      3. the rest are tag
  6. Set associative: gives a limited # of location where data may be placed
    1. 2-way set associative:
      1. For 8bits address
      2. last two which byte in word
      3. last third which word in block
      4. last fourth which set in cache
      5. the rest are tag
  7. Replacement Policies
    1. For direct mapped: no choice, one data one seat
    2. For set associative: 
      1. LRU: least recently used
    3. For fully associative:
      1. Random
      2. FIFO
