1. static instruction scheduling 
    1. move other unrelated instruction before the stalled instruction
  2. Optimization of branches:
    1. delay slots: that are one or more slots after a branch instruction that the hardware will execute whether or not the branch is taken
    2. It’s the responsibility of the compiler to ensure that the instruction in those slots are safe(will execute no matter the branch is taken or not), if they always execute when the branch is taken
    3. the code inside the delay slot will execute no matter the branch is taken or not, such as Store instruction before the branch
    4. if there is no instruction in the slot, put NOOP
  3. All of these are local scheduling
  4. Global scheduling 
    1. Loop unrolling: making multiple copies of a loop before branch
      1. have more duplicate things in my block so I have more thins I can move
      2. Remove the cost of some of the branches
      3. you have to make sure it will end up with the right iteration
      4. Before rescheduling, if the registers are infinite, we don’t need to worry about the hazard=>register renaming
