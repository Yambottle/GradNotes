1.  MIPS
    1. LW R1, #8(R2)
    2. LW dest, #offset(source) fetch then add the offset
  2. Instruction encoding: 3 types
    1. 3 register instruction: 
      1. opcode-6bits, RS-5bits, RT-5bits, RD-5bits(32 register)=>ADD R1, R2, R3
      2. immediate value in instruction=> opcode-6bits, RS-5bits, RT-5bits, immediate value-16bits=>ADD R1, R2, -3=>LW
    2. I-type:
      1. Branch: opcode-6bits, RS-5bits, RT-5bits
    3. J-type:
      1. opcode, target
  3. RISC(Reduced instruction set computing)=> has simple instruction
    1. readable
    2. keep all instructions same length, easier to fetch
    3. easier to pipeline or extract ILP
  4. CISC(Complex instruction set computing)=> complex instruction set
    1. instruction count is less; 
    2. important when don’t have a lot of memory; 
    3. code shorter
    4. we still use it for backwards compatibility
  5. One way to use CISC on RISC hardware is to use microcode(micro-machine will translate for the lower hardware)=>Intel called Microps
  6. LW does most of the work:
    1. fetch the instruction
    2. decode => see what type of instruction you have=>evaluate the opcode and generate the control signals=>figure out which register need to be read
    3. execute=>calculate the address of data by adding RS to the sign extended immediate
    4. memory=>use address to fetch the data
    5. write back
  7. Pipeline lets us have different instructions in different stages
    1. 5 stages are just like above works
    2. ideal speedup should be the pipeline stages
