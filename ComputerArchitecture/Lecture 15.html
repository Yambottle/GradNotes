1. 5-stage pipeline: everything is in order
    1. RAW hazards are solved with forwarding and stall cycles
  2. diversified pipeline: divide floating point and integer into two pipeline
  3. different functions take different amounts of time: then one function for one pipeline
    1. problem: ID is a bottleneck because instructions can’t proceed if they depend on an instruction further in the pipeline and if the dependency can’t be handled with data forwarding, 
      1. we as a compiler can reorder instructions to replace stalls
      2. let computer dynamically reorder
  4. Tomasulo’s Algorithm
    1. We need to worry about RAW, WAW, WAR
      1. we can rename registers but it’s not infinite
    2. Or we use reservation stations for register renaming
      1. break ID into two parts:
        1. Issue: Decode instructions and check for structure hazards and if possible, we place instruction in a reservation station
        2. Read operands: We wait for operand values to be available and then we read them
    3. Component/Field in reservation station
      1. Busy=> is there an instruction in this RS that hasn’t completed execution yet?
      2. Op=> operation to perform for instruction
      3. Qj and Qk=>The RS that will produce the corresponding source operand
      4. Vj and Vk=> corresponding the values of the source operands
        1. This can be loaded directly from the register file
        2. It can come from the common data bus(CDB) when the data in the RS indicated by Qj or Qk is placed on the bus
      5. A=>used to stored the address for memory instructions
      6. Qi is associated with the register file. This corresponds to the RS that will produce the data that will get stored in this register. If Qi=0, then the value in the register file is valid
    4. For lots of RAW
      1. Assumptions: 
        1. only one instruction is issued to a RS per clock
        2. instruction can start executing if operands a functional unit are available
        3. once put in functional unit it takes x clock cycles to finish based on type of instructions
        4. After this happens, data can be placed on CDBus and other RS can read it 
        5. An instruction that was this value can start executing if a functional unit is available
