  1. Diversified pipeline: Video

  1. LATENCY=> is the number of clocks that an immediately following dependent instruction needs to wait in ID to avoid a RAW hazard
    1. Integer ALU: 0 latency
    2. Load: 1 latency
    3. FP operation: 4 latency
    4. => number of cycles spent in execution
  2. Initiation interval=> the minimum number of cycles between issuing 2 instructions of the same type to the execution units
    1. If the functional units are pipelined the initiation interval is 1
      1. Integer ALU: 1 
      2. Load: 1
      3. FP operation: 1 (5 if no pipe) 
  3. FP divide op is hard to pipe, so 24 latency and 25 initiation interval
