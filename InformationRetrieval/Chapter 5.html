  2. Lossless and lossy
  3. Heap’s law: M is size of vocabulary, T is the number of tokens in the collection
    1. empirical finding
    2. M=kT^b
    3. 30<=k<=100, b is about 0.5
  4. Zipf’s law: relative frequencies of terms
    1. the i th frequent term has cf=K/i, K is a normalizing constant
    2. log cf = log K - log i
    3. linear relationship between log cf and log i
  5. Posting list compression
    1. we can use 20 bits per docID
    2. for the, a, an, we can use 0/1 bitmap
    3. we can also store the gaps, if each gap is not too long (less than 20 bits)
    4. Huffman coding
    5. Variable Byte: give bit to byte, cut the first bit(continue or end flag bit)
    6. Gamma code: cut off the first bit of the binary code, and put the length of Unary code before that. 
      1. Unary code: n 1s ends with 0
      2. 13->1101->101->called offset
      3. length 3->1110
      4. gamma code->1110101
      5. always odd length
      6. if the value >15, the gamma code will save space
    7. Variable Byte is simpler
