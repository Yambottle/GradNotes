  3. MySQL: automatically commit, show set auto-commit=0 first, then can rollback
  4. Usage:
    1. Log: within transaction inside to recover DB
    2. Lock: data integrity 
    3. Two-phase Locking
      1. Read: shared lock
      2. Write: exclusive lock
      3. Commit: released all locks
    4. Concurrency  Control: serializable schedules:
      1. Basic: expand and shrink
      2. Conservative: request all until commit
      3. Strict: exclusive locks are not released until commit
      4. Rigorous: lock are only released at the end 
  5. Transaction deadlock: multiple transactions can run at the same time; the solution is set a timeout; and the transaction level
  6. Phantom Reads: one delete, one insert, then select
  7. non-repeatable: when one select twice, during that one insert
  8. dirty read: one read the value before one update
  9. Schedules:
    1. nonrecoverable schedule: not use
    2. strict schedule: other nonRW transactions run after W transactions
