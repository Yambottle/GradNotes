  7. Simplex Method: method for computer to solve LP problems
    1. convex combination
      1. two points x,y belongs to real numbers, a convex combination of x and y is z= l*x+(1-l)y, l is [0,1]; z is the convex combination, between a straight line of x and y 
    2. convex set: a set is a convex iff it contains all convex combinations of any two points within it
    3. concave 
    4. special case:
      1. an empty set or one-single point set are both convex set
      2. the intersection of 2 convex sets is also convex set
    5. convex function: f(lx1+(1-l)x2)<=l f(x1)+(1-x)f(x2)
      1. e.g. f(x) = square(x)
    6. concave function: f(lx1+(1-l)x2)>=l f(x1)+(1-x)f(x2)
    7. Extreme point P: 
      1. condition 1: P is in a convex set
      2. condition 2: For any line segment that contains P and is located wholly in S and P is an endpoint of this line
      3. e.g. three points of triangle
      4. if the convex set S is a polygon, then the extreme points are also called corner points
    8. STD LP formula: 
      1. cares about maximize, because min can convert to max
      2. all constraints should be equations
      3. all decision variables are non-negetive
      4. e.g z=3*x1+2*x2
        1. 2x1+x2<=9  => 2x1+x2+s1=9, s1>=0; use this way to change comparison to equation, s1 is slack variables
        2. 2x1+x2>=9  => 2x1+x2-e1=9, e1>=0; use this way to change comparison to equation, e1 is excess variables
      5. then we can apply it in to all general STD LP problems
      6. STD LP matrix form:
        1. max z= c.T*x
        2. s.t Ax=b, x>0
        3. then computer can do the matrix computation
    9. How computer use simplex method to solve STD LP
      1. brute force all the possibility of all decision variables and slack variables
      2. e.g. max z= 3x1+2x2
        1. s.t. 2x1+x2+s1=9
        2. x1+2x2+s2=9
      3. solutions:
        1. x1=x2=0, s1=s2=9; yes (0,0)
        2. x1=s1=0, x2=9, s2=-9; no
        3. x1=s2=0, x2=4.5, s1=4.5; yes (0,4.5)
        4. x2=s2=0, x1=9, s1=-9;no
        5. x1=s2=0, x1=x2=3; yes (3,3)
        6. x2=s1=0, x1=4.5, s2=4.5; yes (4.5,0)
      4. then pick all feasible one to maximize the results
      5. if we have n variables and m constraints in a STD LP, then try to set any n-m variables as non basic variables which are equal to zero, then serve for the remaining m variables as basic variables.
      6. Any solution obtained from this step, we called it Basic Solutions. 
      7. Any Basic Solutions in which all variables are non-negetive are called Basic Feasible Solutions
      8. Theorem: 
        1. first
          1. The feasible region of any LP is a convex set
          2. B F Solutions are extreme points
          3. If a LP has an optimal solution, there must be an extreme point that is optimal
          4. Not all optimal solutions are extreme points
          5. Not all extreme points are solutions
        2. second
          1. For any LP with m constraints, 2 BFSs are adjacent extreme points if they have m-1 basic variables in common
          2. the # of potential solutions are C(n, n-m); n: variables, m: constraints
      9. Another better solution:
        1. Draw the feasible region
        2. Find an extreme point to the LP and calculate the Basic Value
        3. Check all adjacent points of  current extreme point and calculate their Basic Value
        4. If no adjacent points has larger value, then it is the optimal; otherwise, change the extreme point and go back to step 3
